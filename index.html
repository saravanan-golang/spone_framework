Sponge Framework: A Powerful Solution for Backend Services

What is Sponge Framework?

Sponge is a robust framework designed to simplify the creation of high-performance backend services. By integrating automatic code generation with frameworks like Gin (for web services) and gRPC (for remote procedure calls), Sponge allows developers to focus on business logic rather than boilerplate code.

With the power to generate code directly from SQL or Protobuf files, Sponge streamlines the development process by automating tasks like CRUD operations, API definitions, and business logic templates.

Key Features of Sponge Framework

Automatic Code Generation

Sponge generates code based on your database schema or Protobuf files, taking care of CRUD operations, business logic, and API endpoints.

Microservices Architecture Support

With built-in tools for scaling, replacing components, and monitoring, Sponge provides seamless integration for microservices architectures.

Installation Steps

Getting Started with Sponge

            
# Install Sponge
go install github.com/zhufuyi/sponge/cmd/sponge@latest

# Initialize Sponge and automatically install plugins
sponge init

# Verify plugin installation
sponge plugins --install
sponge plugins

# Check Sponge version
sponge -v

Once installed, start the Sponge UI with:

sponge run

Then access it at http://localhost:24631 in your browser.

Core Design Philosophy

Sponge is a tool designed to simplify backend service development. Its core design philosophy revolves around reverse-generating modular code from SQL or Protobuf files. This method helps developers focus more on business logic by automating the generation of CRUD API services. It eliminates manual coding and allows developers to rapidly deploy backend services with minimal effort, all while maintaining flexibility and scalability.

Key Goals

  • Automatically Generate CRUD API Services: Sponge generates full CRUD APIs for web and gRPC services, freeing developers from manually writing backend code.
  • Simplify General Service Development: Sponge allows for quick development by auto-generating database access code, Protobuf API descriptions, and service templates. Developers only need to focus on business logic.
  • Reduce Development Time and Errors: By automating repetitive tasks, Sponge reduces the chance of human error and speeds up development, ultimately enabling rapid service delivery.

Code Generation Framework

The code generation framework of Sponge is based on two primary methods: SQL and Protobuf. These methods facilitate the creation of various types of backend services, ranging from CRUD-based web services to more complex gRPC-based services. SQL support includes MySQL, MongoDB, PostgreSQL, TiDB, and SQLite databases, while Protobuf allows for web, gRPC, and combined services.

Project Creation Methods

  • Create Web Service Based on SQL: Quickly generate a web service with complete CRUD functionality without writing any Go code.
  • Create gRPC Service Based on SQL: Automatically generate a gRPC service using SQL definitions for CRUD APIs.
  • Create Web Service Based on Protobuf: Create a fully functional web service based on Protobuf definitions.
  • Create gRPC Service Based on Protobuf: Automatically generate gRPC service code from Protobuf API definitions.
  • Create gRPC Gateway Service Based on Protobuf: Generate a gRPC gateway service that enables HTTP and gRPC communication.
  • Create gRPC+HTTP Service Based on Protobuf: A hybrid service that supports both gRPC and HTTP requests, ideal for complex APIs.

Use Case for SQL-based Web Service

This method is ideal for generating full web back-end services, including a complete database layer and CRUD API, which can be deployed with minimal configuration. It's particularly useful for projects like admin panel backends that require standardized, CRUD-based interfaces.

Use Case for Protobuf-based Web Service

This method is ideal for creating robust and scalable web services using Protobuf files for API definitions. It is particularly useful for systems requiring high-performance, strongly typed APIs for communication between services. Protobuf ensures efficient serialization, making it ideal for gRPC and other services that need to handle large volumes of data or maintain cross-language compatibility. It supports both internal and external service communication with the flexibility to define APIs in a structured manner.

Microservices Framework

Sponge also supports a microservices framework that is built to scale. The framework includes several essential components for distributed systems such as web frameworks (Gin), RPC frameworks (gRPC), database ORM (GORM, Mongo-go-driver), and more. It is designed to be flexible and can be easily integrated with other services in a microservices ecosystem.

Key Components:

  • Web Framework: Gin, known for its speed and simplicity in web development.
  • RPC Framework: gRPC, offering high performance for remote procedure calls in distributed systems.
  • Database ORM: GORM and Mongo-go-driver for easy database interaction.
  • Message Queue: RabbitMQ, Kafka for asynchronous messaging.
  • Service Discovery: Nacos, Consul, and etcd for easy service registration and discovery.
  • Continuous Integration: Integration with Jenkins, Docker, and Kubernetes for seamless deployment.

Project Code Directory Structure

Monolithic Application or Microservices Repository

The directory structure for a Sponge-generated project is organized to help manage various aspects of backend service development. Below is an overview of key directories:

  • api: Contains API definitions and Protobuf files.
  • assets: Stores static resources such as images and documentation.
  • cmd: The entry point for the application, including initialization and service registration files.
  • configs: Houses configuration files for the service.
  • docs: Stores API documentation, such as Swagger-generated files.
  • internal: Contains business logic and private code (e.g., database access, routing, models).
  • pkg: Shared libraries used across the service.
  • scripts: Includes build, deployment, and code generation scripts.
  • test: Contains testing files and test SQL scripts.
  • third_party: External dependencies or tools, such as helper programs and forked code.

Sponge Core Features

Template Code

The template code forms the backbone of Sponge projects. It includes predefined structures for web and gRPC services, designed to simplify development. The code is divided into three key areas: Code Generation Tools, Web and gRPC Services Code, and Basic Development Framework Scripts. These parts help developers focus on customizing business logic, while Sponge automatically handles repetitive tasks like API setup and service configuration, allowing for faster, error-free development.

Code Generation

Code generation in Sponge automates the creation of essential service components like CRUD operations, API definitions, and database access logic. Developers can generate service templates through the UI or command-line tools, which will integrate seamlessly into the project. By offering various generation options, such as web services or gRPC services based on SQL or Protobuf, Sponge significantly reduces the time spent on boilerplate code, enhancing productivity and minimizing manual coding errors.

Egg Model for Service Code Generation

The "Egg Model" is a structural framework used by Sponge for service code generation. It divides the project into multiple layers, with core functionality encapsulated in a protective outer shell. This approach ensures that the service code remains modular, maintainable, and scalable. The model emphasizes separation of concerns by organizing different components, such as API definitions, business logic, and database access, into distinct layers. This modular approach simplifies development, future updates, and troubleshooting.
